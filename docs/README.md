# docs — 文档总览

文档按「输入 → 消化 → 产出」的工作流分三层管理，每层一个目录：

| 目录 | 名字含义 | 角色 | 装什么 |
|---|---|---|---|
| [`1-ref/`](1-ref/) | reference，参考 | **输入** | 外部拿进来的原始材料，照搬为主 |
| [`2-exp/`](2-exp/) | explore，探索 | **消化** | 调研/学习/评估的过程，可乱、可推翻 |
| [`3-sol/`](3-sol/) | solution，方案 | **产出** | 已拍板的设计与结论，`sol` = 当前事实 |

## 流转关系

```
ref ──读取──▶ exp ──沉淀──▶ sol
拿进来        嚼透          定下来
```

- 从 `ref` 取材料，在 `exp` 里调研、学习、评估、试错；
- 某个结论确定后，把结论搬进 `sol`，推导过程留在 `exp`——直到所属决策全部拍板，随后按根目录 [housekeeping.md](../housekeeping.md) 清理；
- 结论变更时更新 `sol`，始终保持 `sol` 反映当前事实。

## 怎么放文件

- 拿不准放哪：问自己「这是别人的、我在想的、还是我定了的」——分别对应 `ref` / `exp` / `sol`。
- 各目录的详细约定见对应的 `README.md`。
- 记录门槛与清理准则（内容何时移出工作区）见仓库根目录 [housekeeping.md](../housekeeping.md)。

> 序号 `1/2/3` 只为固定排序、保证目录列表与工作流同序；去掉序号后是 `ref / exp / sol`，首字母 **r / e / s** 对应 **r**eference → **e**xplore → **s**olution 的先后。
