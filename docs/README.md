# docs — 文档总览

文档按「输入 → 消化 → 产出」的工作流分三层管理，每层一个目录：

| 目录 | 角色 | 装什么 |
|---|---|---|
| [`refs/`](refs/) | **输入** | 外部拿进来的原始参考材料，照搬为主 |
| [`notes/`](notes/) | **消化** | 调研/学习/评估的过程笔记，可乱、可推翻 |
| [`design/`](design/) | **产出** | 已拍板的设计与结论，始终等于当前事实 |

## 流转关系

```
refs ──读取──▶ notes ──沉淀──▶ design
拿进来          嚼透            定下来
```

- 从 `refs` 取材料，在 `notes` 里调研、学习、评估、试错；
- 某个结论确定后，把结论搬进 `design`，推导过程留在 `notes`——直到所属决策全部拍板，随后按根目录 [docs-rules.md](../docs-rules.md) 清理；
- 结论变更时更新 `design`，始终保持 `design` 反映当前事实。

## 怎么放文件

- 拿不准放哪：问自己「这是别人的、我在想的、还是我定了的」——分别对应 `refs` / `notes` / `design`。
- 各目录的详细约定见对应的 `README.md`。
- 记录门槛与清理准则（内容何时移出工作区）见仓库根目录 [docs-rules.md](../docs-rules.md)。
