# design — 最终方案

存放已经拍板的产出：设计出了什么、确定了什么、最终方案是什么。这里是「结论」，应当稳定、可信、可直接照着做。

## 装什么

- 最终设计：架构、模块划分、数据结构、接口约定
- 已确定的决策：选了哪个方案、定了哪些规则及理由
- 实施方案：可落地的步骤、配置、规范

## 不装什么

- 外部参考材料 → 放 [`../refs`](../refs)
- 推导过程、被否掉的候选、评估细节 → 放 [`../notes`](../notes)

## 约定

- 只写已确认的结论；探索中、未定的内容留在 [`../notes`](../notes)。
- 重大决策注明理由，必要时链接 [`../notes`](../notes) 里的评估过程或 [`../refs`](../refs) 里的依据。
- 结论变更要更新这里，保持「design = 当前事实」。
