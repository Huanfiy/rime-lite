# notes — 探索消化

记录把问题摸熟的过程：调研了什么、学习了什么、评估了什么。这里是「想的过程」，可以乱、可以推翻。

## 装什么

- 调研笔记：某个技术/方案怎么用、有什么坑
- 学习记录：读完 [`../refs`](../refs) 材料后的理解、消化、推导
- 方案对比与评估：候选项的优劣、打分、权衡
- 实验与验证：试错过程、中间结论、被否掉的想法

## 不装什么

- 外部原始材料 → 放 [`../refs`](../refs)
- 最终拍板的方案与结论 → 放 [`../design`](../design)

## 约定

- 过程性内容，允许不完整、不严谨、随时修改。
- 评估和对比尽量给依据（数据、引用 [`../refs`](../refs) 里的来源），便于日后复盘。
- 一旦某个结论沉淀为最终方案，把结论搬到 [`../design`](../design)，这里保留推导过程。
- 推导过程只保留到所属决策全部拍板；拍板后按仓库根目录 [docs-rules.md](../../docs-rules.md) 清理，移出工作区（git 历史可查）。
