# docs-rules — 记录与清理规则


| 项    | 内容                                            |
| ---- | --------------------------------------------- |
| 创建日期 | 2026-07-07                                    |
| 状态   | 现行（决策 D-16，见 [docs/design/architecture.md](docs/design/architecture.md) §2） |
| 定位   | AI 辅助开发长期迭代中的信息治理规则：明确记录什么、记在哪、何时清理、怎么清理        |
| 不覆盖  | 词库数据维护（见 [docs/design/lexicon-sop.md](docs/design/lexicon-sop.md)）；git 历史管理（历史永不改写） |


## 1. 原则

- 工作区只保留两类信息：**当前事实**（拍板结论、现行配置、现行流程）与**未决过程**（支撑未决决策的推导）。其余归 git 历史。
- 信息以「消费」为生命周期终点：输入材料被消费、决策被拍板、阶段被验收，即触发清理审视（§4）。
- 同一事实只允许一处正文，其余位置以链接指向；历史版本由 git 承担，文档不承担版本职能。
- AI 会话产出信息的速度远高于人工，默认动作是「不记录」；落档需过 §2 的记录门槛。

## 2. 记录框架


| 层    | 位置                | 装什么                       | 生命周期               |
| ---- | ----------------- | ------------------------- | ------------------ |
| 事实速查 | `fact.md`         | 已决事项一行式摘要、当前状态、未决清单       | 随决策 / 阶段变更同轮更新     |
| 对外说明 | `README.md`       | 定位、使用方式、维护入口              | 随功能变更更新            |
| 拍板结论 | `docs/design/`     | 架构、决策记录（D-n 编号）、SOP       | 就地更新，始终等于当前事实      |
| 探索过程 | `docs/notes/`     | 未决决策的推导与评估                | 所属决策全部拍板后清理        |
| 参考输入 | `docs/refs/`     | 将被消费的外部原始材料               | 消费完毕后清理            |
| 变更记录 | git log           | 变更主题、筛选口径、验证结论（一笔一主题）     | 永久，不改写             |


记录门槛（全部满足才落档，否则不记录）：

1. 面向未来读者（下一轮会话、他机、他人），而非面向当前对话；
2. 无法从代码、配置、git 历史直接推导；
3. 有明确归属层，且该层尚无同一事实——已有则更新原文，不另起新文。

## 3. 不记录清单

- 过程性对话、试错细节、被否弃的中间方案（直接支撑某个未决决策的除外，归 `docs/notes/`）。
- 可推导事实：配置内容不复述进文档（引用文件路径），变更历史不复述（git log 即记录）。
- 一次性中间产物：候选 TSV、staging 目录、探针程序与日志——放仓库外（`/tmp` 或会话临时目录），不入库。
- 运行态文件：由 `.gitignore` 隔离（`build/`、`*.userdb/`、`sync/` 等）。
- AI 会话内部状态（任务清单、计划草稿）：`.agents/`、`.claude/` 已隔离，不入库。

## 4. 清理准则

**触发点**（事件驱动，不设固定周期）：

- 阶段验收完成；
- 决策拍板（相关 exp 推导失去「未决」属性）；
- 输入材料消费完毕（refs 归档）；
- 发现文档表述与配置现状冲突（以配置为准修正文档，冲突表述即清理对象）。

**判据三问**（全「否」即清理）：

1. 它描述的还是当前事实吗？
2. 它支撑的决策还未决吗？
3. 它有 git 历史之外、不可再生的价值吗？

**清理方式**：

- 文件级：移出工作区；所有引用处改为「已移出工作区，git 历史 `<引入提交 hash>` 可查」；不留墓碑文件。
- 章节级：正文清空，保留章节号与一行墓碑说明（保持编号稳定，避免既有引用漂移）。
- 决策行（D-n）：被推翻不删行，原地标注「已被 D-m 推翻」——决策链本身是当前事实的一部分。
- 清理独立成笔提交（`🔧 chore`），提交说明写明清理对象与理由。
